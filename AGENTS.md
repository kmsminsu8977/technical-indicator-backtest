# AGENTS.md

## 역할

이 저장소에서 작업하는 에이전트는 **기술적 지표 백테스트** 주제를 금융공학 연구 노트와 재현 가능한 코드베이스로 정리한다. 모든 설명, 커밋 메시지 초안, 문서 본문은 기본적으로 한국어로 작성한다.

## 먼저 읽을 기준 문서

- `AGENTS.md`: 이 저장소에서 작업할 때 지켜야 할 역할, 품질 기준, 산출물 규칙
- `SOURCE.md`: `KAIST-DFMBA` 폴더에서 어떤 경로와 아이디어를 참고해야 하는지 정리한 프로젝트별 source inventory
- 새 컨텐츠를 만들거나 기존 컨텐츠를 고칠 때는 반드시 `SOURCE.md`의 참고 범위와 재구성 기준을 먼저 확인한다.

## 공통 목적

- `KAIST-DFMBA` 로컬 폴더의 과목별 자료 흐름을 참고해 금융공학 연구를 체계화한다.
- 단순 과제 모음이 아니라 연구 질문, 이론 배경, 데이터 처리, 모형 구현, 검증 결과가 이어지는 저장소로 유지한다.
- 참고 자료의 아이디어는 이 저장소의 주제에 맞춘 독립적인 설명, 합성 예제, 재현 가능한 실험으로 재구성한다.

## 작업 기준

- `SOURCE.md`의 참고 자료와 연구 질문을 먼저 확인하고 작업 범위를 정한다.
- 노트북은 탐색과 설명에 사용하고, 반복 가능한 핵심 로직은 `src/`로 분리한다.
- 데이터는 `data/raw`, `data/processed`, `data/sample`로 구분한다.
- 산출물은 `outputs/charts`, `outputs/images`, `outputs/tables`에 저장하며, 생성 조건을 문서나 코드에 남긴다.
- 실험 결과를 보고할 때 표본 기간, 입력 변수, 파라미터, 비용 가정, 평가 지표를 함께 적는다.

## 현재 프로젝트 초점

- 카테고리: Market Signals & Event Analytics
- 핵심 질문: 이동평균 교차 신호는 거래비용을 고려해도 단순 보유보다 나은 경로를 만드는가?
- 구현 위치: `src/`
- 입력 위치: `data/sample/`
- 결과 위치: `outputs/tables/`

## 품질 기준

- 금융 개념은 수식, 직관, 구현 순서로 설명한다.
- 백테스트나 예측 결과는 룩어헤드 바이어스, 생존편향, 과최적화, 거래비용을 점검한다.
- 확률 모형과 시뮬레이션은 난수 시드, 경로 수, 수렴성 진단을 명시한다.
- 포트폴리오 연구는 수익률뿐 아니라 변동성, 최대낙폭, 샤프비율, 회전율, 리스크 기여도를 함께 검토한다.
- 텍스트/NLP 연구는 수집 시점, 토큰화 방식, 라벨 또는 감성 점수의 기준을 기록한다.

## 금지 사항

- 출처와 재구성 기준 없이 외부 내용을 붙여넣지 않는다.
- 검증되지 않은 투자 성과를 확정적 표현으로 쓰지 않는다.
- API 키, 계정 정보, 유료 데이터 원본, 개인정보가 포함된 파일을 커밋하지 않는다.
- 생성된 중간 파일과 캐시는 필요한 경우를 제외하고 저장소에 남기지 않는다.

## 권장 산출물

- `README.md`: 연구 개요와 실행 방법
- `SOURCE.md`: 참고 경로, 재구성 기준, 현재 산출물 연결
- `docs/methodology.md`: 이론 배경, 모형 가정, 검증 방법
- `notebooks/`: 탐색 분석과 결과 설명
- `src/`: 재사용 가능한 데이터 처리, 모형, 평가 함수
- `outputs/`: 표, 그림, 차트
